# Malligai EPS Registration

A registration portal for **Malligai EPS** with a public form, UPI payment step
and a branded admin console. Built with a React + Vite + Tailwind frontend and a
dependency-free PHP backend (MySQL/MariaDB).

## What it does

**Public page (`/`)**

- Branded splash screen using the JCI logo and the site theme.
- Registration form collecting **Name, Phone No (WhatsApp), Mail ID, Date of
  Birth, Address** — all mandatory.
- UPI QR payment step: pay the registration fee and upload the payment
  screenshot in **any image format** (JPG, PNG, HEIC, WEBP, GIF, BMP…).
- **Call** and **WhatsApp** buttons to contact the team about EPS registration.

**Admin console (`/admin`)**

- Summary tiles (total registrations, today, total collected) and a searchable
  table of every submission.
- **View** each registration with all details and a preview + download of the
  payment screenshot.
- **Download Excel** (.xlsx) and **Download PDF** — both fully branded with the
  logo/theme and every field.
- **Send ID Card** — generates a JCI-branded EPS ID card (PNG) with all details
  and the paid confirmation, then shares it to the registrant's WhatsApp (native
  file share on mobile; download + WhatsApp chat on desktop).

## Tech

- **Frontend**: React 19, Vite, Tailwind CSS v4, axios.
- **Backend**: plain PHP 8 (PDO/MySQL). Excel and PDF are generated by small
  self-contained libraries in `backend/lib/` — no Composer required. The ID card
  is rendered with the GD extension.

### Server requirements

PHP 8+ with the `pdo_mysql`, `gd`, `zip` and `fileinfo` extensions, plus
MySQL/MariaDB. If `zip` is unavailable the Excel export automatically falls back
to CSV.

## Setup

### 1. Database

```bash
mysql -u root -e "CREATE DATABASE epsjci CHARACTER SET utf8mb4;"
mysql -u root epsjci < database/eps_registrations.sql
```

### 2. Backend

```bash
cd backend
cp .env.example .env      # then edit .env
php -S localhost:5000 -t .
```

Key `.env` settings:

| Variable                | Purpose                                              |
| ----------------------- | ---------------------------------------------------- |
| `DB_*`                  | MySQL connection                                     |
| `ADMIN_USERNAME/PASSWORD` | Admin console login                                |
| `ADMIN_TOKEN_SALT`      | Random string; changing it logs admins out           |
| `REGISTRATION_FEE`      | Fee shown on the form and printed on the ID card     |
| `CONTACT_PHONE`         | Number for the **Call** button (dialled as-is)       |
| `CONTACT_WHATSAPP`      | Digits only (country code + number) for **WhatsApp** |

### 3. Frontend

```bash
cd frontend
npm install
npm run dev        # dev server on http://localhost:5173
npm run build      # production build into dist/
```

`frontend/.env.development` / `.env.production` set `VITE_API_URL` (the backend
base URL).

## Deployment (shared hosting)

1. Build the frontend (`npm run build`) and upload `dist/` to the web root.
2. Upload `backend/` alongside it and create `backend/.env`.
3. Ensure `backend/uploads/payments/` is writable by PHP.
4. The included `.htaccess` files route the React app and expose clean API paths
   while blocking direct access to `config/`, `lib/`, `.env` and the raw uploads
   (payment images are only served through the token-protected admin endpoint).

## Optional: ID card font

The ID card uses a bundled system sans-serif by default. To pin the typeface,
drop `Regular.ttf` and `Bold.ttf` into `backend/assets/fonts/`.
